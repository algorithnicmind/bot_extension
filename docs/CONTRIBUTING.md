# Contributing to the AI Assistant Chrome Extension

First off, thank you for considering contributing to this incredible AI Assistant Extension! It's people like you that make the open-source community such a powerful place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs

If you find a bug or issue, please create an Issue on the GitHub repository. Provide as much detail as possible:

- A clear and descriptive title.
- Steps to reproduce the issue.
- The exact error message (if any).
- Your browser version and OS.

### Suggesting Enhancements

Have a great idea for a new AI feature or an improvement to the UI? We'd love to hear it!

- Open an Issue using the "Feature Request" tag.
- Explain _why_ this enhancement would be useful to most users.
- Provide mockups or examples if applicable.

### Pull Requests

We actively welcome your pull requests! To contribute code:

1.  **Fork the repo** and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  If you've changed APIs or features, update the documentation in the `docs/` folder.
4.  Ensure your code follows the existing style and is clean.
5.  Issue that pull request!

## Setting Up the Project Locally

Please refer to `docs/3_setup_guide.md` for full instructions on how to load the unpacked extension into Chrome for localized testing.

## Code of Conduct

This project and everyone participating in it is governed by a Code of Conduct. By participating, you are expected to uphold this code. Please report unacceptable behavior to the repository owner.

Happy contributing! 🚀
